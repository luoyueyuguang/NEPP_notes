# 408 计算机学科专业基础综合讲义审阅报告

审阅日期：2026-07-22

## 1. 结论先行

经过多轮整改、逐题复审和专项反证检查后，讲义的四科主干知识、章节组织、章节习题解析和三套模拟卷结构已达到可使用状态；截至本报告更新时能够确认的知识性错误、答案硬伤、多答案/无正确选项、题干条件不足、解析缺失、模型混用和致命构建问题均已修正或撤下。

当前正式内容与截至审阅日可确认的、稳定施行的 408 科目框架及近年统考试卷结构一致：

- 40 道单项选择题，每题 2 分，共 80 分；
- 7 道综合应用题，共 70 分；
- 满分 150 分，考试时间 180 分钟；
- 数据结构 45 分、计算机组成原理 45 分、操作系统 35 分、计算机网络 25 分。

需要同时保留三个边界：

1. 本报告不能证明整套材料“绝无错误”。结论是：截至当前复审确认出的硬伤已清除，正式内容中的章节习题已形成答案闭环，静态检查和完整构建通过，并建立了后续发布门槛。
2. 原有精确真题年份/题号没有完成逐卷权威核验，已从正式讲义中隔离；正文题目统一按“真题风格”发布，不能把它们当作正式真题索引。
3. 截至审阅日不能把材料表述为“已与尚未发布的 2027 新大纲完全一致”。2027 新大纲发布后仍须做一次增量复核。

综合判断：**可以作为当前 408 系统复习讲义、章节训练和模拟训练使用；章节题目与解析已形成闭环，模拟卷结构、科目分值、主干覆盖和基本题风相符，每套又补入 4 道中等或多步骤选择题，难度分布较首轮明显改善，但仍不应宣传为与某一年度正式卷完全等难。**

### 2.2 审阅基准

- **范围基准**：数据结构、计算机组成原理、操作系统、计算机网络四科的稳定主干框架；
- **结构基准**：40 道单选 + 7 道综合应用题，150 分，180 分钟，四科分值 45/45/35/25；
- **题型基准**：单选题强调概念辨析、条件判断、过程模拟和小规模计算；综合题强调多步骤推导、算法/过程描述和跨知识点建模；
- **质量基准**：题干条件充分、答案唯一、选项可判定、解析与答案一致、公式和代码边界明确、图表不依赖缺失素材；
- **发布基准**：不得把未经逐卷核验的年份、题号和题源归属作为事实发布。

### 2.3 审阅方法

本轮采用仓库级审阅，而不是只抽查少量页面：

1. 对四科高风险计算、算法过程、协议状态和系统综合题重新推演；
2. 检查选择题是否存在多答案、无正确答案或条件不足；
3. 检查综合题题干信息是否足以推出答案，解析是否覆盖关键步骤；
4. 核对三套模拟卷的题数、选择题答案数、分值和四科配比；
5. 扫描 TODO、待核实、草稿式自我否定、未经核验的年份/题号和过强断言；
6. 连续完整构建并检查致命错误、交叉引用、缺字、未定义命令和严重横向溢出。
7. 第三轮额外按章节统计正式习题与解析标记，检查解析开关是否被子章节重新定义，并对新增解析中的算法、状态序列和数值结果做独立复算。

这套方法能够显著降低知识和工程质量风险，但不能替代官方大纲发布后的逐条对照，也不能替代精确真题元数据的逐卷查验。

## 4. 本轮主要知识性修订

以下是影响正确性或学习边界的代表性修订，不是逐行变更日志。

### 4.1 数据结构

- 补齐数组地址计算、行/列优先存储、特殊矩阵和稀疏矩阵；
- 修正 Huffman 合并代价、编码一致性、WPL 与概率未归一化时的解释；
- 修正 AVL 插入序列的旋转类型和最终树形；
- 修正遍历重构题的唯一答案，删除“查原题”等草稿痕迹；
- 修正拓扑序、Dijkstra、AOE 网和关键路径相关题解；
- 统一 B 树/B+树术语，修正 B 树高度、删除过程和散列表 ASL；
- 明确二次探测覆盖结论的适用形式与模数条件；
- 明确图遍历序列的唯一性依赖起点和邻接点扫描顺序；
- 将链表高效排序改为自底向上归并排序。

### 4.2 计算机组成原理

- 补齐 ASCII、8421 BCD、大小端、边界对齐、奇偶校验、海明码、SEC-DED 和多类移位；
- 修正补码、IEEE 754、Booth 乘法、流水线、Cache、微程序编码和扩展操作码题解；
- 修正 DRAM 集中刷新、分散刷新和异步刷新的定义；
- 不再把精确 LRU 说成高相联硬件中普遍采用的实现，补充伪 LRU/随机替换边界；
- Cache 映射统一为直接映射、组相联和全相联三类；
- 区分 Fault、Trap、Abort 的返回语义，不再把异常笼统称为“软件中断”；
- 修正 DMA 周期挪用、总线竞争和“完全并行”的绝对化表述；
- 修正指令周期表述：间址和中断周期并非每条指令必然经历；
- 对除零异常的 Fault/Abort 分类增加体系结构边界；
- 将多处宽表和长公式改成可换行布局。

### 4.3 操作系统

- 补齐线程模型、IPC、Peterson、TestAndSet、Swap、CAS 和管程；
- 修正银行家算法资源总量、Need/Available 计算和安全性判断；
- 对 Max/Need 信息不完整的综合题明确结论为“信息不足”，不再强行给出安全序列；
- 重新计算页面访问串：3 个页框时 OPT=11、FIFO=16、LRU=15，4 个页框时 FIFO=14；该串不构成 3→4 页框的 Belady 异常；
- 明确缺页有效访问时间公式中缺页处理时间是否包含重新执行后的访存；
- 修正动态分区最佳适应算法的局部判据，不再声称其全局外部碎片最小；
- 修正 FAT 随机访问表述，区分链接分配与索引分配；
- 修正字符设备缓存、DMA、单/双缓冲和多类 PV 题；
- 修正多实例资源分配图的剩余资源和可化简序列；
- 修正 i-node 最大文件大小、二级间接区偏移和磁盘平均访问时间；
- 明确 `MAP_SHARED` 与 `MAP_PRIVATE` 的写回/COW 语义；
- 改写错误的跨模块进程—页面—I/O 时间线，列出完成此类题所需的全部时序条件。

### 4.4 计算机网络

- 补齐停止—等待、GBN、SR、ALOHA、CSMA、CSMA/CA、802.11、隐藏站、RTS/CTS、VLAN、NAT/NAPT/PAT 和私网地址；
- 修正以太网最短帧、子网划分、HTTP RTT、DNS 递归/迭代、SMTP/MIME/端口和 NAT 校验和；
- 区分 408 教材化拥塞窗口规则与 TCP Reno 快速恢复规则；
- DNS `hosts` 文件不再被称为 DNS 缓存；
- CSMA/CD 的短帧问题改为“无法保证发送期间检测到冲突”，不再误说成“无法重传”；
- 曼彻斯特编码的跳变方向明确以题目约定为准，并限定经典 10 Mb/s 以太网语境；
- 修正 IPv4 分类前缀位、DSCP/ECN、TTL 初始值和 IPv6/NAT 的绝对化表述；
- 修正 POP3/IMAP 的服务器副本、文件夹和离线缓存语义；
- 删除或弱化会迅速过时的固定介质速率/距离数字，保留稳定的比较关系。

## 5. 第二轮独立复审新增发现与处理

第二轮没有把首轮结论当作既定事实，而是重新检查高风险定义、适用条件、算法复杂度、系统过程题和三套模拟卷答案。新增发现和整改如下。

### 5.1 数据结构

- 循环链表只能方便地从当前结点沿一个方向循环访问，任意查找仍通常为 \(O(n)\)；删除“任意位置快速到达任意位置”的错误表述。
- 子孙结点定义明确排除结点自身，避免把“后代”和“自身或后代”混用。
- 任意二叉树的非递归遍历辅助空间统一表述为 \(O(h)\)，只有在树高 \(h=O(\log n)\) 时才可写成 \(O(\log n)\)。
- 原并查集代码实际按集合大小合并，却标为“按秩合并”；现已区分 rank 与 size，并将代码、变量和解释统一为按大小合并。
- Shell 排序复杂度依赖增量序列和输入分布，删除无条件“平均约 \(O(n^{1.3})\)”的说法。
- 模拟卷完全二叉树算法改为输入规模决定的动态队列容量，补全结点计数函数和提前返回时的内存释放；LCA 题明确两个目标结点均存在且互异。

### 5.2 计算机组成原理

- 摩尔定律只用于描述集成度的历史趋势，不再把晶体管数量增长直接等同于处理器性能同比例增长。
- 删除 SRAM/存储层次中过时的固定纳秒数字，改为稳定的相对速度和容量关系。
- 存储层次不再被描述成任何实现中都严格满足“上层内容必为下层子集”；高位交叉编址不能提高的范围限定为连续单地址流。
- 流水线冒险题补充常规转发通路前提，避免在硬件条件未说明时断言某类相关“一定”阻塞。
- 修正状态/控制寄存器复用对独立可寻址寄存器和端口数量的影响。
- DMA 周期挪用不再写成“只在 CPU 不访存时偷偷传输”；DRAM 地址复用题补充行、列地址位数相同的条件；扩展操作码、Cache 和流水线综合题均重新推演。

### 5.3 操作系统

- 首次适应只保留其搜索和实现特征，不再无条件称为“综合性能最好”。
- 分页仍需检查页号、页表项有效性和访问权限；删除“不需要越界检查”的绝对说法，只说明它不做分段式段长—偏移比较。
- 中断驱动 I/O 的“一次一个字节/字”限定为 408 简化模型，实际控制器可能使用 FIFO 或突发传输。
- 原题把“SCAN（含 LOOK）”写在题干中却只解 LOOK，现已明确为 SSTF 与 LOOK。
- i-node 最大文件大小精确修正为 \(4\,\mathrm{GiB}+4\,\mathrm{MiB}+32\,\mathrm{KiB}=4,299,194,368\,\mathrm{B}\)，并修正二级间接路径与 I/O 次数解释。
- 章节末 10 道综合练习原有无答案、条件不足和过程缺失；现已全部补齐或改写，并独立复算页表、银行家、FIFO/LRU/CLOCK、磁盘调度、PV、i-node、TLB/Cache 和跨模块时间线。

### 5.4 计算机网络

- 电路交换总时延不再被缩减为“仅传播时延”，明确建立、发送和传播时延，以及它与存储转发的差异。
- CRC 检出全部单比特错误补充生成多项式的必要条件。
- IPv4 分片补充 DF=0/DF=1 边界，并明确 IPv6 中间路由器不执行分片。
- 重新区分 IGMP 与组播路由协议的职责，不再笼统声称“局域网组播不需要 IGMP”。
- “IP 地址+端口号”不再无条件称为唯一网络端点；补充传输层协议以及 TCP 四元组语境。
- 主机到本地 DNS 的递归查询改为典型教学模型/服务器策略；HTTP/1.1 明确默认持久连接，关闭连接使用 `Connection: close`。

### 5.5 模拟卷质量升级与独立复算

三套卷各替换 4 道基础直问，共新增 12 道中等或多步骤选择题：

- 卷一：Huffman WPL、Cache 地址划分、完整单级页表大小、HTTP 持久连接流水线；
- 卷二：图的边数与生成树、DDR 理论带宽、TLB 有效访问时间、GBN 信道利用率；
- 卷三：Dijkstra、理想流水线总时间、分页物理地址、子网与 ARP 联合判断。

独立复算的代表性结果包括：Huffman WPL=228，Cache Tag=19 位，TLB 有效访问时间=120 ns，GBN 利用率约 19.05%，100 条指令的 5 段流水线时间=208 ns，物理地址=5122，FIFO（3/4 页框）缺页数=16/14，LRU（4 页框）=10，CLOCK（4 页框）=14，i-node 最大字节数=67,379,200，CRC 余数为 `000`。这些复算结果均与修订后的解析一致。

选择题答案位置也做了分布审计和选项重排；只调整选项顺序并同步答案，不改变知识点。最终分布为：卷一 A/B/C/D=9/11/10/10，卷二=9/11/10/10，卷三=10/10/10/10，消除了原先正确答案明显集中于 B、D 极少的问题。

## 6. 第三轮逐题闭环复审新增发现与处理

第三轮重点不再停留于主干抽查，而是检查“每道正式章节习题是否可独立作答、是否有答案或解析、解析是否与题干模型一致”。本轮对 9 个原本解析不完整的章节进行逐题补齐：这些章节共有 112 道习题，其中新增 110 个原先缺失的答案/解析；配合原有 2 个答案后，已全部形成闭环。其余章节也重新统计，正式四科各章均满足题目数与解析数一致。

### 6.1 习题工程与发布开关

- 补齐 `ds/chap2`、`ds/chap3`、`ds/chap7`、`ds/chap8`、`co/chap1`、`co/chap3`、`co/chap4`、`os/chap1`、`os/chap5` 的缺失解析；选择、填空、算法、计算和综合题均给出可核验步骤，而非只列答案字母。
- 发现 5 个子章节重复声明并强制执行 `\showsolutionstrue`，导致主入口即使设置隐藏解析，后续章节仍会自行重新打开。现已移除局部声明，解析开关只在 `main.tex` 定义和控制。
- 静态计数确认全部正式章节习题有对应解析；`co/chap2`、`co/chap6` 等采用不同章节标题的文件也单独核查，没有因脚本标题匹配差异而漏审。

### 6.2 数据结构

- 栈的首个出栈元素题原允许 `k=1`，会使“1 位于栈底”的选项失去普遍成立性；现限定为 (2\le k\le n)。
- KMP 题明确 next/nextval 的 1 起始下标约定，修正 `T="aaab"` 的 nextval 题原有两个正确选项，并把“只计匹配”与“构造 next + 匹配”的复杂度口径分开。
- BF 平均复杂度不再脱离概率模型直接写成 (O(n+m))；`abcabca` 的 next/nextval 和 `ababcabcacbab` 匹配过程已逐比较复算。
- 快排一趟划分、小根堆、大根堆连续删除、Shell 各趟结果和 3 路最佳归并树均重新模拟；3 路归并 WPL=213，按记录读/写计的总 I/O 量为 426。
- 快慢指针中点题删除语义重复选项并明确循环条件。后续反证复审进一步澄清：对固定整数步长，判环阶段应依据相对位移同余式判断；慢1快3仍必会在有限环内相遇，但不能直接照搬慢1快2的入口定位结论。

### 6.3 计算机组成原理

- 修正性能题的过约束矛盾：原题同时给出 CPI 比、时钟周期比和目标时间，三者不相容；现改为同指令条数、CPI 比和目标时间，唯一结果为 4 GHz。
- Cache 映射题原为“正确的是”却有 3 个正确选项，已改为唯一错误项；连续访存题统一按字编址并补足传送粒度和额外开销假设。
- 精确 LRU 的状态位题明确询问“区分 (k!) 种全序的理论最小编码”，答案为 (\lceil\log_2(k!)\rceil)，4 路时为 5 位；不再与具体年龄计数器或伪 LRU 位数混写。
- 低位/高位交叉编址、按列访问 Cache、TLB + L1 + L2 延迟和虚拟地址到 Cache 的题均补足模型条件；串行计时、淘汰假设和高位交叉落体条件均写入题干。
- 扩展操作码题原选项中没有正确的 128，且题干未给地址字段位数；现明确两个地址字段各 6 位并提供正确选项。另一道所谓 16 位指令格式原字段合计仅 10 位，已重写成总计 16 位的 `OP|Md|Rd|Ms|A/Rs` 格式并补全编码约定。
- 收紧虚拟存储、Cache 缺失、DRAM 固定时延、单级页表和可嵌套中断的绝对化表述，区分 408 教材模型与具体体系结构实现。

### 6.4 操作系统

- 删除“没有中断就没有进程切换”的绝对说法，补充系统调用、异常、主动让出和阻塞也可触发切换；并发/并行题补充并行硬件语境。
- 单道/多道批处理题补足单 CPU、非抢占 FCFS、到达顺序、I/O 可彼此并行并与 CPU 重叠等条件；复算结果为单道 210 ms、多道 130 ms。
- 多级反馈队列在未给周期性提升/老化时不能保证无饥饿，修正原解析中“通过老化防饥饿但不保证无饥饿”的自相矛盾。
- 2 KiB 双缓冲题补足每批 2 块且三个阶段均线性增长的模型。1 KiB 单/双缓冲均为 62.20 ms，2 KiB 双缓冲为 62.40 ms；稳态均为 0.31 ms/块，不能把边界差异误说成吞吐率提升。
- 系统调用改称应用主动请求内核服务的“规范入口”；64 位单级稠密页表改称空间代价不现实，而不是数学意义上的“不可能”。

### 6.5 计算机网络

- TCP 超时后的慢开始题原选项答案为 `1,2,4,8,9`，解析却写成 `1,2,4,6,7`。现明确“每轮开始时判断 cwnd 与 ssthresh”的教材化规则，答案与解析统一为 `1,2,4,8,9`。

- 接收窗口为 10 KiB 的题区分 cwnd 与实际发送窗口：拥塞窗口仍可继续增长，真正受限的是 (\min(\mathrm{cwnd},\mathrm{rwnd}))，不能说 cwnd 达到 rwnd 后停止增长。

## 17. 第六轮正确性阻断项整改（2026-07-22）

第六轮针对“知识定义、语言级未定义行为、系统过程边界、综合题条件闭环”重新审查，没有沿用上一轮的通过结论。复审确认 4 项会阻止零硬伤验收的问题，并已全部修正：

1. **树的结点度数之和**：按数据结构中的定义，结点的度是孩子数，总和应为 $n-1$；只有把树视为无向图并统计图论顶点度时，总和才是 $2(n-1)$。正文现已分开定义并消除前后矛盾。
2. **原码加减交替除法 C 示例**：原实现对可能为负的 `int64_t R` 执行左移，属于 C 语言未定义行为。现改为在 $1\le n\le32$ 的显式契约下使用 `2 * R + bit`，增加除数、位宽、输入范围和输出指针检查，并用布尔返回值区分成功与非法参数。
3. **DMA 响应时机与数据通路**：DMA 总线请求改为在当前总线周期结束的边界响应，中断请求通常在当前指令周期结束后响应；数据通路统一表述为不经过 CPU 通用寄存器、不由 CPU 指令逐字搬运，同时说明具体 DMAC 可以包含数据缓冲寄存器。
4. **模拟卷二第 46 题缺页处理**：题设中的 4 个页框均已占用，解析不再写“分配空闲页框”，而是补全合法性检查、选择牺牲页、脏页写回、旧页表项/TLB 失效、调页、更新映射和重新执行指令的流程。由于题目未给置换算法，解析明确不能唯一确定淘汰哪一页。
